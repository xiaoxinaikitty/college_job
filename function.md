# 大学生实习求职系统功能文档（精简闭环版）

## 1. 文档目标
本版本只定义“可上线、可运营、可追踪”的最小闭环功能，确保系统能完整跑通：
学生投递 -> 企业处理 -> 面试 -> Offer -> 录用/淘汰 -> 评价/举报 -> 平台审核与治理。

## 2. 系统角色与端
- 学生端（Flutter App）
- 企业端（Flutter App）
- 管理后台（Vue3）
- 后端服务（SpringBoot + MySQL）

## 3. 核心业务闭环
1. 学生注册并完善简历。
2. 企业认证并发布岗位（管理员审核后上架）。
3. 学生搜索岗位并投递。
4. 企业查看投递并与学生沟通。
5. 企业发起面试并记录结果。
6. 企业发放Offer，学生接受或拒绝。
7. 流程结束后学生可评价企业，异常可举报。
8. 管理员处理审核、举报与处罚，形成治理闭环。

## 4. 功能范围（MVP）

### 4.1 学生端（必须）
- 账号：注册/登录/退出。
- 简历：创建、编辑、选择默认简历。
- 岗位：列表、搜索、筛选、详情查看。
- 投递：一键投递、投递记录、状态跟踪。
- 沟通：与企业在线消息沟通。
- 面试/Offer：查看面试安排、查看并处理Offer（接受/拒绝）。
- 反馈：评价企业、举报岗位/企业。

### 4.2 企业端（必须）
- 企业账号：注册登录、企业资料维护。
- 企业认证：上传资质，等待审核。
- 岗位管理：发布、编辑、下线岗位。
- 候选人管理：查看投递、筛选、状态流转（待处理/沟通中/面试/Offer/淘汰/录用）。
- 沟通与面试：消息沟通、发起面试、填写面试结果。
- Offer：发放Offer、查看学生处理结果。

### 4.3 管理后台（必须）
- 管理员登录与权限控制（基础RBAC）。
- 企业资质审核（通过/驳回）。
- 岗位审核（通过/驳回/下架）。
- 用户管理（学生/企业状态管理）。
- 举报处理（受理、结案、处罚记录）。
- 基础运营看板（注册量、岗位量、投递量、面试量、Offer量、录用量）。

### 4.4 后端服务（必须）
- 统一鉴权：JWT 登录态与角色鉴权。
- 用户中心：学生、企业、管理员账户体系。
- 简历服务：简历增删改查。
- 岗位服务：岗位发布、查询、审核状态流转。
- 投递服务：投递、去重、状态流转日志。
- 沟通服务：会话与消息。
- 面试服务：面试安排与反馈。
- Offer服务：发放、接收、拒绝、过期处理。
- 审核举报服务：审核、举报、处罚闭环。
- 通知服务：站内通知（投递状态、面试、Offer、审核结果）。

## 5. 关键状态流转（必须落库）

### 5.1 岗位状态
草稿 -> 待审核 -> 已上线 -> 已下线/已驳回

### 5.2 投递状态
已投递 -> 已查看 -> 沟通中 -> 面试中 -> Offer阶段 -> 已录用  
已投递/沟通中/面试中 -> 已淘汰  
已投递 -> 已撤回（可选）

### 5.3 Offer状态
已发放 -> 已接受 / 已拒绝 / 已过期

### 5.4 举报状态
待处理 -> 处理中 -> 已结案

## 6. 最小数据对象
- 用户（学生/企业/管理员）
- 企业与企业认证
- 岗位
- 简历
- 投递记录与状态日志
- 会话与消息
- 面试安排与反馈
- Offer
- 评价与举报
- 审核与处罚记录
- 通知消息

## 7. 非功能要求（精简）
- 安全：密码加密、接口鉴权、权限校验、敏感信息脱敏。
- 性能：核心查询与投递接口稳定可用，支持分页与索引。
- 可追踪：关键操作写日志（审核、状态变更、处罚）。
- 可运维：统一错误码、统一日志、基础监控告警。

## 8. 上线验收标准（闭环判定）
- 学生可从“注册-投递-面试-Offer-评价”完整走通。
- 企业可从“认证-发岗-处理候选人-发Offer”完整走通。
- 管理员可完成“审核-举报处理-处罚-数据查看”完整走通。
- 所有关键状态流转可追溯（有时间、有操作者、有结果）。

## 9. 二期能力（本期不做）
- 智能推荐与算法排序
- 职业测评/课程/活动运营
- 高级数据分析与BI报表
- 多渠道消息（短信/邮件）与复杂风控策略
